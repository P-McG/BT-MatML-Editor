The Mat\+ML Editor is a text Editor for the \href{https://www.matml.org/}{\texttt{ Mat\+ML 3.\+1 XML Schema}}.

Mat\+ML is an extensible markup language (XML) developed especially for the interchange of material information. It provides a medium of communication for users in materials science and related fields such as manufacturing and aerospace. It sets the stage for the development of semantic web standards to enhance knowledge discovery in materials science and related areas.\hypertarget{index_autotoc_md0}{}\doxysection{Binary Downloads}\label{index_autotoc_md0}
Goto \href{https://github.com/P-McG/BT-MatML-Editor/releases}{\texttt{ Releases}} and under {\itshape Assets} download the win64 for Microsoft Windows releases and Linux for Linux releases.\hypertarget{index_autotoc_md1}{}\doxysection{BT-\/\+Mat\+ML-\/\+Editor Usage}\label{index_autotoc_md1}
To run the program, click on the BT-\/\+Mat\+ML-\/\+Editor application.

Once running click on \textquotesingle{}File-\/\texorpdfstring{$>$}{>}Open\textquotesingle{}, or \textquotesingle{}File-\/\texorpdfstring{$>$}{>}New\textquotesingle{}, to start.

The general approach to the program is to use the Tree Control user interface to navigate through the Mat\+ML data elements. The tree on the left side is the main navigation aid (1). Mouse click on the elements of the tree and then left mouse click to get more options, such as to add more data elements to the Tree Control. The Mat\+ML Schema Information will show up on the right panel (2). The tabs for the right panel (3) will display either the Mat\+ML Schema Information (default) or the Mat\+ML data itself (can be changed in \textquotesingle{}Edit-\/\texorpdfstring{$>$}{>}Preferences\textquotesingle{}).



While the BT-\/\+Mat\+ML-\/\+Editor is not a database, and not optimized for extremely large files, it can handle multimaterial files. These larger files are more manageable using the following tools.

Sorting the view of the materials. In \textquotesingle{}Edit-\/\texorpdfstring{$>$}{>}Preferences-\/\texorpdfstring{$>$}{>}Sort class\textquotesingle{}, it is possible to add sort classes. This option will changed the view (not the xml file) of the materials. Adding \char`\"{}\+STEEL\char`\"{} for an example, will display the Materials with the class name of \char`\"{}\+STEEL\char`\"{} first. Then the remainding materials will be displayed in the order the are in inside the XML file. Once set, \textquotesingle{}view-\/\texorpdfstring{$>$}{>}perform material sort\textquotesingle{} will change the view order of the materials.

One other option for viewing larger files is the use of \textquotesingle{}view-\/\texorpdfstring{$>$}{>}Isolate material branch\textquotesingle{}. This will isolate materials which are interconnected with \textquotesingle{} \mbox{\hyperlink{class_material}{Material}} -\/\texorpdfstring{$>$}{>} \mbox{\hyperlink{class_component_details}{Component\+Details}} -\/\texorpdfstring{$>$}{>} \mbox{\hyperlink{class_class}{Class}} -\/\texorpdfstring{$>$}{>} \mbox{\hyperlink{class_parent_material}{Parent\+Material}} \textquotesingle{} associations. To exit this view, \textquotesingle{}View-\/\texorpdfstring{$>$}{>}Exit Isolate material branch\textquotesingle{} will bring back the original list of materials.\hypertarget{index_autotoc_md2}{}\doxysection{BT-\/\+Mat\+ML-\/\+Editor Builds}\label{index_autotoc_md2}

The program is structure to perform the following tasks\+:
\begin{DoxyEnumerate}
\item Parsing the Mat\+ML XML Schema using Xerces-\/c and binding the XML data into C++ using Code\+Synthesis-\/\+XSD.
\item Create GUIs using wx\+Widgets.
\item Polymorphically bind the Mat\+ML XML data to the wx\+Widgets GUIs.
\end{DoxyEnumerate}\hypertarget{index_autotoc_md6}{}\doxysubsection{Parsing the Mat\+ML XML Schema using Xerces-\/c and binding the XML data into C++ using Code\+Synthesis-\/\+XSD.}\label{index_autotoc_md6}
The XML data binding into C++ is done using Code\+Synthesis-\/\+XSD data binding compiler. The compiler produces the \mbox{\hyperlink{matml31_8hxx_source}{matml31.\+hxx}}, matml31.\+cxx files. Add these to your project file.

Place the XML Schema Definition file matml31.\+xsd in the same directory as the xml data ($\ast$.xml).

The xerces-\/c parsing libraries should be linked in your project. For Windows adding xerces-\/c\+\_\+3\+\_\+1.\+dll and xerces-\/c\+\_\+3\+\_\+1\+D.\+dll to the library paths would do it.

With these files above, the user can perform C++ material calculations.

The Mat\+ML\+\_\+\+Editor is the GUI for the purpose of providing an input, or output, to these C++ material calculations or used on its own as a Mat\+ML Editor.\hypertarget{index_autotoc_md7}{}\doxysubsection{Create GUIs using wx\+Widgets}\label{index_autotoc_md7}
As per the wx\+Widgets DECLARE\+\_\+\+APP and IMPLEMENT\+\_\+\+APP macros, class \mbox{\hyperlink{classbellshire_1_1_b_t___mat_m_l___app}{bellshire\+::\+BT\+\_\+\+Mat\+ML\+\_\+\+App}} defines the entry point for the console application.

\mbox{\hyperlink{classbellshire_1_1_b_t___mat_m_l___app_a88e6aede9e4dd98b1793f84883872310}{bellshire\+::\+BT\+\_\+\+Mat\+ML\+\_\+\+App\+::\+On\+Init}} function calls for the creation of the class \mbox{\hyperlink{classbellshire_1_1_material_frame}{bellshire\+::\+Material\+Frame}}, which is the main GUI class for the application.

\mbox{\hyperlink{classbellshire_1_1_material_frame}{bellshire\+::\+Material\+Frame}} constructor \mbox{\hyperlink{classbellshire_1_1_material_frame_base_aeed67876cae3e4d072f6272407ae2ab3}{bellshire\+::\+Material\+Frame\+Base\+::\+Material\+Frame\+Base}} then calls for the creation of all the different Mat\+ML GUI classes via the class \mbox{\hyperlink{classbellshire_1_1_g_u_i}{bellshire\+::\+GUI}}.

The class \mbox{\hyperlink{classbellshire_1_1_g_u_i}{bellshire\+::\+GUI}} constructor \mbox{\hyperlink{classbellshire_1_1_g_u_i_a8a0da888fba0f022d360e3769712c327}{bellshire\+::\+GUI\+::\+GUI(wx\+Window$\ast$ parent)}}, calls for the creation of the individual Mat\+ML GUIs, such as class \mbox{\hyperlink{classbellshire_1_1_mat_m_l___doc___g_u_i}{bellshire\+::\+Mat\+ML\+\_\+\+Doc\+\_\+\+GUI}}.

The individual Mat\+ML GUIs, such as class \mbox{\hyperlink{classbellshire_1_1_mat_m_l___doc___g_u_i___base}{bellshire\+::\+Mat\+ML\+\_\+\+Doc\+\_\+\+GUI\+\_\+\+Base}} constructor \mbox{\hyperlink{classbellshire_1_1_mat_m_l___doc___g_u_i___base_a0d928b23f428c42d3a658574e19721ad}{bellshire\+::\+Mat\+ML\+\_\+\+Doc\+\_\+\+GUI\+\_\+\+Base\+::\+Mat\+ML\+\_\+\+Doc\+\_\+\+GUI\+\_\+\+Base}}, then create the final GUI for the associated Mat\+ML data class, such as \mbox{\hyperlink{class_mat_m_l___doc}{Mat\+ML\+\_\+\+Doc}}. See the table below for the Mat\+ML data and the associated Mat\+ML GUI classes.

The base classes, such as \mbox{\hyperlink{classbellshire_1_1_material_frame_base}{bellshire\+::\+Material\+Frame\+Base}} and the individual Mat\+ML GUIs, such as \mbox{\hyperlink{classbellshire_1_1_mat_m_l___doc___g_u_i___base}{bellshire\+::\+Mat\+ML\+\_\+\+Doc\+\_\+\+GUI\+\_\+\+Base}}, contain the creation of the GUIs, virtual Event Handler Functions, and binding the Event Handler Functions.

The derived classes, such as \mbox{\hyperlink{classbellshire_1_1_material_frame}{bellshire\+::\+Material\+Frame}} and the individual Mat\+ML GUIs, such as \mbox{\hyperlink{classbellshire_1_1_mat_m_l___doc___g_u_i}{bellshire\+::\+Mat\+ML\+\_\+\+Doc\+\_\+\+GUI}}, contain the overwriting of the Event Handler Functions occuring in the base.

It should also be noted that not all {\itshape instances} of Mat\+ML data have Mat\+ML GUIs associated with them, but all {\itshape classes} of Mat\+ML data do. This means that changing Mat\+ML data will require showing, and updating, the Mat\+ML GUIs to show the new Mat\+ML data.

Because the not all {\itshape instances} of Mat\+ML data have Mat\+ML GUIs associated with them, but all {\itshape classes} of Mat\+ML data do, the Mat\+ML Editor relies on polymorphic functions to select the appropriate class type to operate on.

The polymorphic functions uses boost\+::any pointers. The \mbox{\hyperlink{classbellshire_1_1_tree_ctrl_sorted}{bellshire\+::\+Tree\+Ctrl\+Sorted}}, a derivative of wx\+Tree\+Ctrl, stores user data in each of its tree elements. Using this functionality the editor stores data such as the \mbox{\hyperlink{classbellshire_1_1_mat_m_l_tree_item_data}{bellshire\+::\+Mat\+MLTree\+Item\+Data}} which contains the boost\+::any pointer.

The \mbox{\hyperlink{classbellshire_1_1_tree_ctrl_sorted}{bellshire\+::\+Tree\+Ctrl\+Sorted}} mimics the tree structure of the Mat\+ML\+\_\+data and allows the user data to be pulled from the tree elements and used.

After the user data is pulled the a given Mat\+ML data\`{}s class typeid is compared to the boost\+::any.\+type. When the comparison matches, a function is either called, or the data is boost\+::any\+\_\+cast back into the Mat\+ML data pointer for further processing.

When the Mat\+ML data structure is changed the \mbox{\hyperlink{classbellshire_1_1_tree_ctrl_sorted}{bellshire\+::\+Tree\+Ctrl\+Sorted}} is recreated once again mimicking the Mat\+ML data structure.

An example is a function for updating of the Mat\+ML GUIs,

void \mbox{\hyperlink{classbellshire_1_1_g_u_i_af44e953baa7788a7ff32461ea93395ee}{bellshire\+::\+GUI\+::\+Update\+\_\+\+GUI(\+::boost\+::any anyptr, \+::boost\+::shared\+\_\+ptr$<$\+Mat\+ML\+\_\+\+Doc$>$ doc)}}

this will Update the appropriate GUI depending on the type of Mat\+ML data pointer the \+::boost\+::any pointer caste into.

Another, function is a function showing the context menu,

void \mbox{\hyperlink{classbellshire_1_1_material_frame_a62dbd938705e88cfdafa26b00d5dd7a7}{bellshire\+::\+Material\+Frame\+::\+Show\+Context\+Menu(wx\+Tree\+Item\+Id id, const wx\+Point\& pt)}}

it will create a context menu showing possible selections depending on the \mbox{\hyperlink{classbellshire_1_1_mat_m_l_tree_item_data}{bellshire\+::\+Mat\+MLTree\+Item\+Data}} (has a member of boost\+::any pointer) selected.
